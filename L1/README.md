Лабораторная работа 1:


Что я сделал
1. Написал программу на C
Я реализовал вычисление факториала (двумя способами: итеративно и рекурсивно) и чисел Фибоначчи (итеративно). Программа принимает число n из командной строки, если ничего не ввели — считает для n=10.

2. Получил ассемблерный код с разными оптимизациями
Я выполнил команды:

bash
gcc -S -o main_O0.s main.c -O0   # без оптимизации
gcc -S -o main_O1.s main.c -O1   # базовая
gcc -S -o main_O2.s main.c -O2   # агрессивная
gcc -S -o main_O3.s main.c -O3   # максимальная
gcc -S -o main_Os.s main.c -Os   # по размеру
Получил 5 файлов .s и увидел, как код меняется в зависимости от флагов.

3. Разобрался с ассемблером (-O2) и добавил комментарии
Я взял main_O2.s и подробно закомментировал каждую строку. Нашел:

Циклы — метки .L3, .L15, переходы jne, jle

Переменные — они живут в регистрах: %eax, %ecx, %edx, %r8d, %r9d

Вызовы функций — call printf, call atoi

Понял, что в -O2 нет обращений к стеку, всё в регистрах — поэтому код работает быстрее.

4. Переделал программу в модульную структуру
Я разбил код на три файла:

fibonacci.h — заголовок с объявлениями функций

fibonacci.c — реализация (факториалы и фибоначчи)

main.c — только точка входа

Также сгенерировал ассемблер для каждого модуля отдельно (main.s и fibonacci.s).

5. Разработал Makefile
Я написал Makefile, который автоматизирует сборку:

makefile
make        # собирает программу
make clean  # удаляет временные файлы
make run    # запускает
make asm    # генерирует ассемблер
6. Добавил параллельные процессы (Windows)
Я создал программу parallel.c, которая:

Запускает два параллельных процесса через CreateProcess

Использует мьютекс (CreateMutex, WaitForSingleObject, ReleaseMutex) для синхронизации

Пишет результаты в общий файл result.txt

Оба процесса одновременно вычисляют факториал и число Фибоначчи, но мьютекс не дает им мешать друг другу при записи в файл.

Что я понял и узнал:
[ Что узнал  | Как это работает ]
Регистры	%eax — результат, %ecx — 1-й аргумент, %edx — счетчик
Цикл в ассемблере	метка → тело → add → cmp → jne (прыжок назад)
Оптимизация -O2	переменные в регистрах, нет пролога/эпилога, выравнивание .p2align
Разница -O0 и -O2	-O0: всё на стеке, много лишнего; -O2: всё в регистрах, код короче
Мьютекс	блокирует доступ к ресурсу, пока один процесс не закончит
Вызов функции	аргументы в %ecx, %edx, %r8, %r9, результат в %eax
